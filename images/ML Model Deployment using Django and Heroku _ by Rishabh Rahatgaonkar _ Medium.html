In the previous article we learnt how to train a sentimental analysis model from scratch in python.

Now, in this article we will learnt how to deploy the trained sentimental analysis model using Django and Heroku Cloud platform.

But before that first, lets have a quick introduction to what Django and Heroku is:-

Django is a free open source python web framework which is used to develop websites faster and easier. Django follows MTV(Model Template View ) architecture pattern which makes it fast and easy for development and deployment. Now lets understand briefly what these three terms Model,Template and View means:-

  1. Model- Model represents shape of the data and business logic. It maintains the data of the application. Model objects retrieve and store model state in a database.
  2. Template- The Template is a presentation layer which handles User Interface part completely.
  3. View- View is a user interface. View display data using model to the user and also enables them to modify the data.

Heroku is a container based cloud Platform as a service(PaaS).Developers use heroku to develop,run and operate web applications directly on the cloud platform.It supports various programming languages like Python,Node js,Php etc.

Now we know about both Django and Heroku ,so without any further do lets jump right into our deployment phase.So follow along with me!!!.

Step 1- Creating a Virtual Environment

if you have already created a virtual environment then you can skip this step.

if not then open your terminal and type the following commands to create the virtual environment

conda create --name myEnv django

In the myEnv you can give any name whatever you want.But I have given myEnv name as MyDjangoEnv.

Now Type the following command to activate the environment

activate MyDjangoEnv

Step 2- Creating project and app folder

Create an empty folder wherever you want and give it a name.I would call it McDonalds ML Django Web App.

Now open you favourite text editor and set your text editor path to the project folder to that empty folder by(clicking on the open folder).I will be using visual studio code but you can use other text editors as well.

Now open using favourite terminal (cmd ,windows powershell) or any other terminal inside the folder.

Now type the following command in the terminal to create project root folder.

I called the project folder as mysite but you can give name of your choice

django-admin startproject mysite

this will create a project root folder in your directory

then type

cd mysite

to change your current directory inside the project folder

then type the following command to create your app.

python manage.py startapp mymodel

I named my app as mymodel but you can give name of your choice

Step 3- Running the web app on the local host server.

After activating the virtual environment.

Type the following commands after it.

makingmigrations command tells Django that youâ€™ve made some model changes, and you want to save them as a migration.
